# 数据库启用说明

适用对象：
- 后端同学
- 前端同学
- 算法同学

当前项目统一使用以下本地开发配置。

## 1. 统一配置

数据库：

- 数据库类型：MySQL 8.x
- 主机：`localhost`
- 端口：`3306`
- 数据库名：`coldchain_mvp`
- 用户名：`coldchain_user`
- 密码：`coldchain_dev_123`

后端：

- 服务地址：`http://localhost:18081`
- Spring Profile：`mysql`

算法服务：

- 服务地址：`http://localhost:5001`

前端：

- 开发地址：`http://localhost:18080`

## 2. 建库与建用户

如果本机还没有数据库，请先执行：

```sql
CREATE DATABASE coldchain_mvp DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'coldchain_user'@'localhost' IDENTIFIED BY 'coldchain_dev_123';
GRANT ALL PRIVILEGES ON coldchain_mvp.* TO 'coldchain_user'@'localhost';
FLUSH PRIVILEGES;
```

如果用户已存在，可跳过创建用户，只保留授权。

## 3. 导入表结构与初始数据

执行顺序：

1. `backend-service/sql/schema.sql`
2. `backend-service/sql/seed.sql`

说明：

- `schema.sql` 用来建表
- `seed.sql` 用来导入基础车辆、遥测、告警、风险、路径建议等演示数据

如果重复导入 `seed.sql` 遇到唯一键冲突，说明数据之前已经导入过。

## 4. 后端如何配置

后端本地配置文件：

- [application-local.example.yml](E:/codex_project/JavaWebproject/backend-service/application-local.example.yml)

复制为：

- `backend-service/application-local.yml`

内容统一写成：

```yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/coldchain_mvp?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: coldchain_user
    password: coldchain_dev_123

app:
  algorithm:
    base-url: http://localhost:5001
```

IDEA 启动参数：

```text
--spring.profiles.active=mysql
```

注意：

- 当前这个参数写在“程序参数”位置时，要用 `--spring.profiles.active=mysql`
- 不要在那个输入框里写 `-Dspring.profiles.active=mysql`

## 5. 算法如何配置

算法服务当前默认配置即可，无需额外改数据库连接。

算法只需要保证：

- 服务正常启动在 `5001`
- `POST /evaluate` 可访问
- `GET /simulation/telemetry` 可访问

算法服务当前主要职责：

- 生成模拟遥测数据
- 分析遥测数据并输出风险评估、异常检测、路径建议

## 6. 前端如何配置

前端不直接连数据库。

前端只需要：

- 启动 Vite 开发服务 `18080`
- 通过代理访问后端 `18081`

前端依赖的后端主要接口：

- `GET /api/v1/vehicles`
- `GET /api/v1/vehicles/{vehicleCode}/telemetry/latest`
- `GET /api/v1/dashboard/vehicles/{vehicleCode}`
- `GET /api/v1/vehicles/{vehicleCode}/alerts`
- `GET /api/v1/vehicles/{vehicleCode}/risk-assessments/latest`
- `GET /api/v1/vehicles/{vehicleCode}/route-plans/latest`

## 7. 算法数据接入后端

当前系统已经支持两步手动同步：

1. 导入算法生成的遥测数据

```powershell
Invoke-RestMethod -Method Post http://localhost:18081/api/v1/simulation/import-telemetry
```

2. 导入算法分析结果

```powershell
Invoke-RestMethod -Method Post http://localhost:18081/api/v1/simulation/import-analysis
```

说明：

- 第一步会把算法 `DataGenerator` 生成的遥测写入 `telemetry_records`
- 第二步会基于当前遥测调用算法 `/evaluate`，并把结果写入：
  - `risk_assessments`
  - `route_plans`

## 8. 推荐启动顺序

建议每次本地联调都按这个顺序：

1. 启动 MySQL
2. 启动算法服务 `algorithm-service`
3. 启动后端服务 `backend-service`
4. 如需使用算法生成数据，执行：
   - `import-telemetry`
   - `import-analysis`
5. 启动前端 `frontend`

## 10. 当前项目的数据职责

后端：

- 管数据库
- 管接口
- 管落库
- 给前端提供统一访问入口

算法：

- 生成模拟遥测数据
- 计算异常、风险、路径建议

前端：

- 不直接碰数据库
- 只通过后端接口展示数据

当前本地联调口径已经统一为：

- 数据库账号密码固定
- 后端固定读 `coldchain_mvp`
- 算法固定跑 `5001`
- 前端固定连后端 `18081`

这份说明可以直接发给前端和算法同学照着配。
