# 本地开发与启动说明

适用范围：

- 后端开发
- 前端联调
- 算法联调

## 1. 项目目录说明

当前仓库主要目录：

- `frontend/`
  - 前端页面与静态原型

- `backend-service/`
  - Java Spring Boot 后端项目

- `docs/`
  - PRD 与协作文档

## 2. 后端开发环境要求

后端项目要求：

- JDK：`17`
- 构建工具：`Maven`
- 推荐 IDE：`IntelliJ IDEA`
- 数据库：当前可先不启用 MySQL，默认走 mock 模式

当前后端 Maven 入口文件：

- `backend-service/pom.xml`

当前后端启动类：

- `backend-service/src/main/java/com/coldchain/backend/ColdChainBackendApplication.java`

## 3. IntelliJ IDEA 打开方式

推荐方式：

1. 打开仓库根目录 `JavaWebproject`
2. 确认 IntelliJ 已识别 `backend-service/pom.xml`
3. 在右侧 Maven 面板执行一次 Reload
4. 为模块 `backend-service` 指定 `JDK 17`
5. 运行 `ColdChainBackendApplication`

如果根目录识别不稳定，也可以直接：

1. 用 IntelliJ 打开 `backend-service/pom.xml`
2. 以 Maven 项目导入
3. 再运行主启动类

## 4. 后端启动方式

### 4.1 在 IntelliJ 中启动

直接运行：

- `com.coldchain.backend.ColdChainBackendApplication`

### 4.2 用脚本启动

在仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\backend-service\scripts\start.ps1
```

指定端口启动：

```powershell
powershell -ExecutionPolicy Bypass -File .\backend-service\scripts\start.ps1 -Port 18081
```

说明：

- 启动脚本会优先使用 `mvnw.cmd`
- 如果本机 PATH 中有 `mvn`，会直接调用
- 如果都没有，脚本会提示你安装 Maven 或用 IDEA 导入项目
- 当前默认数据源模式为 `mock`
- 如需切到 MySQL，需要先配置数据库连接，并把 `app.datasource.mode` 改成 `mysql`

## 5. 启动成功后的验证地址

后端启动后，请至少验证下面 4 个地址：

- 首页：`http://localhost:18081/`
- 健康检查：`http://localhost:18081/health`
- 接口文档：`http://localhost:18081/api-docs`
- 车辆列表：`http://localhost:18081/api/v1/vehicles`

如果以上都正常，说明后端运行基本正确。

进一步建议再验证：

- 风险评估：`http://localhost:18081/api/v1/vehicles/CC-VA-01/risk-assessments/latest`
- 路径建议：`http://localhost:18081/api/v1/vehicles/CC-VA-01/route-plans/latest`
- 算法状态：`http://localhost:18081/api/v1/algorithm/status`

## 6. 前端联调方式

前端暂不需要修改后端项目文件。

前端联调时建议：

- 后端地址固定为：`http://localhost:18081`
- 页面优先使用大屏聚合接口：
  - `/api/v1/dashboard/vehicles/CC-VA-01`

如果要做局部刷新，再调用：

- `/api/v1/vehicles/CC-VA-01/telemetry/latest`
- `/api/v1/vehicles/CC-VA-01/alerts?limit=4`
- `/api/v1/vehicles/CC-VA-01/alerts/summary`
- `/api/v1/vehicles/CC-VA-01/risk-assessments/latest`
- `/api/v1/vehicles/CC-VA-01/route-plans/latest`

## 7. 算法联调方式

当前后端仍使用 mock 数据作为主演示数据源。

后续算法接入建议：

- 算法服务独立运行
- 后端通过 HTTP/JSON 调用算法
- 前端不要直接请求算法服务

## 8. 当前主演示数据约定

为了保证演示稳定，当前主用：

- 主场景：疫苗冷链配送
- 主演示车：`CC-VA-01`

原因：

- 这辆车的数据最完整
- 包含升温趋势、预测告警和高风险告警
- 最适合做课堂演示和答辩展示

## 10. 常见问题

### 10.1 IntelliJ 提示模块没有 SDK

处理方式：

- 给 `backend-service` 模块指定 `JDK 17`

### 10.2 项目能打开但不能运行

优先检查：

- 是否导入了 `backend-service/pom.xml`
- 是否配置了 JDK 17
- 是否安装了 Maven

### 10.3 浏览器打开接口只有 JSON 没有页面

这是正常的。

- `/api/v1/...` 是接口，返回 JSON
- `/` 和 `/api-docs` 才是页面入口

### 10.4 前端页面还是假数据

说明：

- 前端还没切换成真实接口
- 需要按 `docs/接口联调说明.md` 对接后端接口
