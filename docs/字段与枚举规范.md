# 字段与枚举规范

适用范围：

- 前端
- 后端
- 算法

目标：

- 统一字段命名
- 统一时间格式
- 统一枚举值
- 降低联调返工成本

## 1. 基础命名原则

- JSON 字段统一使用小驼峰：`vehicleCode`
- 布尔字段统一使用语义明确命名：`doorOpen`
- 经纬度字段固定使用：`lng`、`lat`
- 列表字段使用复数：`alerts`、`temperatureHistory`
- 时间字段名称中尽量带时间语义：`recordTime`、`triggerTime`

## 2. 核心字段字典

### 2.1 车辆字段

- `vehicleCode`
  - 类型：`string`
  - 示例：`CC-VA-01`
  - 说明：车辆业务编号，前后端统一主键字段

- `plateNumber`
  - 类型：`string`
  - 示例：`京A-1024`
  - 说明：车牌号

- `cargoType`
  - 类型：`string`
  - 示例：`VACCINE`
  - 说明：货物类型枚举值

- `cargoName`
  - 类型：`string`
  - 示例：`疫苗`
  - 说明：货物中文名称

- `status`
  - 类型：`string`
  - 示例：`IN_TRANSIT`
  - 说明：车辆状态

### 2.2 温度与轨迹字段

- `recordTime`
  - 类型：`string`
  - 格式：`yyyy-MM-dd HH:mm:ss`
  - 示例：`2026-05-18 09:25:00`

- `temperature`
  - 类型：`number`
  - 单位：摄氏度 `°C`
  - 示例：`7.5`

- `humidity`
  - 类型：`number`
  - 单位：百分比 `%`
  - 示例：`70`

- `doorOpen`
  - 类型：`boolean`
  - 示例：`true` / `false`
  - 说明：车门是否开启

- `speed`
  - 类型：`number`
  - 单位：`km/h`

- `outsideTemp`
  - 类型：`number`
  - 单位：摄氏度 `°C`

- `lng`
  - 类型：`number`
  - 说明：经度

- `lat`
  - 类型：`number`
  - 说明：纬度

- `remainingKm`
  - 类型：`number`
  - 单位：公里 `km`

- `trend`
  - 类型：`string`
  - 示例：`逼近上限`
  - 说明：当前温度趋势描述

### 2.3 告警字段

- `alertId`
  - 类型：`string`
  - 示例：`ALT-20260518-001`

- `level`
  - 类型：`string`
  - 可选值：`LOW`、`MEDIUM`、`HIGH`

- `title`
  - 类型：`string`

- `detail`
  - 类型：`string`

- `suggestion`
  - 类型：`string`

- `triggerTime`
  - 类型：`string`
  - 格式：`yyyy-MM-dd HH:mm:ss`

### 2.4 风险评估字段

- `riskScore`
  - 类型：`number`
  - 说明：风险分数

- `riskLevel`
  - 类型：`string`
  - 可选值：`LOW`、`MEDIUM`、`HIGH`

- `riskLabel`
  - 类型：`string`
  - 示例：`高风险`

- `riskReason`
  - 类型：`string`
  - 说明：风险原因摘要

- `predictedMinutesToLimit`
  - 类型：`number`
  - 说明：预计距离超出安全阈值的分钟数

- `algorithmVersion`
  - 类型：`string`

- `algorithmSource`
  - 类型：`string`
  - 示例：`MOCK_GATEWAY`

### 2.5 路径建议字段

- `planType`
  - 类型：`string`
  - 示例：`REROUTE_COLD_STORAGE`

- `planTitle`
  - 类型：`string`

- `planDetail`
  - 类型：`string`

- `estimatedCost`
  - 类型：`string`

- `estimatedBenefit`
  - 类型：`string`

- `recommended`
  - 类型：`boolean`

- `createdTime`
  - 类型：`string`
  - 格式：`yyyy-MM-dd HH:mm:ss`

### 2.6 大屏聚合字段

- `scene`
  - 类型：`string`
  - 示例：`疫苗冷链配送`

- `systemTime`
  - 类型：`string`
  - 格式：`yyyy-MM-dd HH:mm:ss`

- `safeRange`
  - 类型：`object`
  - 字段：
    - `min`
    - `max`

- `summary`
  - 类型：`object`
  - 说明：顶部概览卡片数据

- `route`
  - 类型：`object`
  - 说明：地图与路径建议数据

- `latestRiskAssessment`
  - 类型：`object`
  - 说明：当前最新风险评估结果

- `temperatureHistory`
  - 类型：`array`
  - 说明：温度趋势图数据

- `alerts`
  - 类型：`array`
  - 说明：告警面板数据

## 3. 时间格式规范

### 3.1 统一格式

所有完整时间统一为：

- `yyyy-MM-dd HH:mm:ss`

示例：

- `2026-05-18 09:25:00`

### 3.2 图表短时间

仅用于前端图表展示时，可使用：

- `HH:mm`

示例：

- `09:25`

说明：

- 后端原始时间字段仍优先返回完整时间
- 前端如要展示短时间，可基于完整时间转换

## 4. 单位规范

- 温度：摄氏度 `°C`
- 湿度：百分比 `%`
- 速度：`km/h`
- 里程：`km`

说明：

- 当前项目中不要混用英制单位
- 前端展示时单位文案必须与此保持一致

## 5. 枚举规范

### 5.1 货物类型

- `VACCINE`
  - 中文：疫苗

当前 MVP 只允许使用这一种主场景货物类型。

### 5.2 车辆状态

- `IN_TRANSIT`
  - 中文：运输中

- `IDLE`
  - 中文：空闲

- `OFFLINE`
  - 中文：离线

当前 mock 数据主要使用：

- `IN_TRANSIT`

### 5.3 告警等级

- `LOW`
  - 中文：低风险

- `MEDIUM`
  - 中文：中风险

- `HIGH`
  - 中文：高风险

说明：

- 后端接口中保留英文枚举更稳定
- 前端页面展示时可映射成中文

### 5.4 告警类型

- `TREND_WARNING`
  - 中文：趋势预警

- `PREDICTION_WARNING`
  - 中文：预测预警

- `DOOR_EVENT`
  - 中文：车门事件

### 5.5 风险等级

当前建议固定：

- `LOW`
- `MEDIUM`
- `HIGH`

对应中文：

- 低风险
- 中风险
- 高风险

如果后续算法输出风险分数：

- 后端统一负责把分数映射到风险等级
- 前端不自行定义映射规则

## 6. 推荐前端映射规则

前端页面展示可采用：

- `LOW` -> `低风险`
- `MEDIUM` -> `中风险`
- `HIGH` -> `高风险`

建议颜色：

- `LOW` -> 绿色
- `MEDIUM` -> 橙色
- `HIGH` -> 红色

## 7. 禁止事项

- 不要把 `vehicleCode` 和 `vehicleId` 混用
- 不要把 `doorOpen` 改成 `0/1`
- 不要把时间格式写成时间戳后又不说明单位
- 不要让前端和算法各自维护不同的风险等级命名
- 不要在不同接口中对同一字段使用不同名字

## 8. 字段变更规则

如果字段需要变更：

1. 先确认是否影响前端和算法
2. 提前同步给相关同学
3. 修改接口代码
4. 同步更新：
   - `backend-service/API.md`
   - `docs/字段与枚举规范.md`
   - 如有必要，更新 `docs/接口联调说明.md`
